As of **May 6, 2026**, I’m ranking **hosted API routes** rather than only base model families, because the same model can be free on one provider with very different throttling on another.

I’m treating **connections/hour** as **requests per hour (RPH)**. When a provider publishes only RPM, I calculate **burst RPH = RPM × 60**, but the **daily cap often throttles you first**.

A few shared rate-limit rules:

* **Google Gemini API:** Google measures limits by **RPM, TPM, and RPD**, but the active free limits are shown in **AI Studio** and can vary by model, tier, preview status, and project. I’m not inventing a fixed Google RPH number because Google says active values should be checked in AI Studio. ([Google AI for Developers][1])
* **OpenRouter `:free` models:** **20 RPM = 1,200 RPH burst**, with **50 requests/day** for a free account with under $10 purchased credits, or **1,000 requests/day** after at least $10 of credits have been purchased. ([OpenRouter][2])
* **GitHub Models free tier:** Low-tier models get **15 RPM / 150 RPD**, high-tier models get **10 RPM / 50 RPD**. That is **900 RPH burst** for low tier and **600 RPH burst** for high tier, but the daily cap is usually the real limit. ([GitHub Docs][3])
* **Cerebras:** publishes RPM, RPH, RPD, TPM, TPH, and TPD per model. ([Cerebras Inference][4])
* **Groq:** publishes RPM, RPD, TPM, and TPD per model; I calculate burst RPH from RPM. ([GroqCloud][5])

I am **not** including Gemini 3.1 Pro Preview because Google’s pricing page shows free-tier input/output as **not available** for that model; this list is free-hosted API only. ([Google AI for Developers][6])

# API key setup + example calls

## 1) Google Gemini API

Use for:

```text
gemini-3-flash-preview
gemini-2.5-pro
gemini-2.5-flash
```

Key setup URL:

```text
https://aistudio.google.com/apikey
```

Setup:

```bash
export GEMINI_API_KEY="YOUR_GEMINI_API_KEY"
```

Example call:

````bash
MODEL="gemini-2.5-pro"
# Other strong free coding options:
# MODEL="gemini-3-flash-preview"
# MODEL="gemini-2.5-flash"

curl "https://generativelanguage.googleapis.com/v1beta/models/${MODEL}:generateContent" \
  -H "x-goog-api-key: ${GEMINI_API_KEY}" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [
      {
        "parts": [
          {
            "text": "You are a senior software engineer. Review this Python code for bugs, edge cases, complexity, and security issues. Then provide a corrected implementation and tests:\n\n```python\ndef group_by_user(events):\n    out = {}\n    for e in events:\n        out[e[\"user_id\"]].append(e)\n    return out\n```"
          }
        ]
      }
    ]
  }'
````

For Google rate limits, open:

```text
https://aistudio.google.com/usage?tab=rate-limit
```

Google documents that Gemini API limits are enforced by RPM, TPM, and RPD per project, and that active tier limits are visible in AI Studio. ([Google AI for Developers][1])

## 2) OpenRouter

Use for:

```text
minimax/minimax-m2.5:free
poolside/laguna-m.1:free
qwen/qwen3-coder:free
nvidia/nemotron-3-super-120b-a12b:free
inclusionai/ling-2.6-1t:free
tencent/hy3-preview:free
openai/gpt-oss-120b:free
z-ai/glm-4.5-air:free
```

Key setup URL:

```text
https://openrouter.ai/keys
```

Setup:

```bash
export OPENROUTER_API_KEY="YOUR_OPENROUTER_API_KEY"
```

Example call:

```bash
MODEL="qwen/qwen3-coder:free"
# Other strong options:
# MODEL="minimax/minimax-m2.5:free"
# MODEL="poolside/laguna-m.1:free"
# MODEL="nvidia/nemotron-3-super-120b-a12b:free"
# MODEL="openai/gpt-oss-120b:free"
# MODEL="z-ai/glm-4.5-air:free"

curl -X POST "https://openrouter.ai/api/v1/chat/completions" \
  -H "Authorization: Bearer ${OPENROUTER_API_KEY}" \
  -H "Content-Type: application/json" \
  -d "{
    \"model\": \"${MODEL}\",
    \"messages\": [
      {
        \"role\": \"system\",
        \"content\": \"You are an expert coding agent. Prefer correct, minimal, testable changes.\"
      },
      {
        \"role\": \"user\",
        \"content\": \"Implement a TypeScript topological sort function with cycle detection. Include Jest tests and explain the time complexity.\"
      }
    ],
    \"temperature\": 0.2
  }"
```

OpenRouter also has a key/limit endpoint you can use to inspect your account:

```bash
curl "https://openrouter.ai/api/v1/key" \
  -H "Authorization: Bearer ${OPENROUTER_API_KEY}"
```

OpenRouter’s `:free` model routes are limited to **20 RPM**, and the free daily cap is usually **50 requests/day** unless the account has purchased at least $10 in credits. ([OpenRouter][2])

## 3) GitHub Models

Use for:

```text
openai/gpt-4.1
deepseek/deepseek-v3-0324
mistral-ai/codestral-2501
```

Key setup URL:

```text
https://github.com/settings/personal-access-tokens
```

Setup:

1. Create a GitHub personal access token.
2. Give it the **Models** permission, commonly shown as `models:read`.
3. Export it:

```bash
export GITHUB_TOKEN="YOUR_GITHUB_PAT"
```

GitHub’s quickstart says local usage requires a PAT with model access, and GitHub Models’ free tier is rate-limited. ([GitHub Docs][21])

Example call:

```bash
MODEL="openai/gpt-4.1"
# Other options:
# MODEL="deepseek/deepseek-v3-0324"
# MODEL="mistral-ai/codestral-2501"

curl -L -X POST "https://models.github.ai/inference/chat/completions" \
  -H "Authorization: Bearer ${GITHUB_TOKEN}" \
  -H "Content-Type: application/json" \
  -d "{
    \"model\": \"${MODEL}\",
    \"messages\": [
      {
        \"role\": \"system\",
        \"content\": \"You are a careful coding assistant. Be precise, test-driven, and security-aware.\"
      },
      {
        \"role\": \"user\",
        \"content\": \"Write a Go HTTP middleware that rate-limits requests by API key using Redis. Include tests and explain failure modes.\"
      }
    ],
    \"temperature\": 0.2
  }"
```

GitHub’s free-tier caps are:

```text
High tier: 10 RPM / 600 RPH burst / 50 RPD
Low tier:  15 RPM / 900 RPH burst / 150 RPD
```

GPT-4.1 and DeepSeek-V3-0324 are high tier; Codestral 25.01 is low tier in GitHub’s catalog/rate-limit tables. ([GitHub][16])

## Best practical stack

For the **hardest coding tasks**, start with:

```text
1. gemini-3-flash-preview
2. gemini-2.5-pro
3. minimax/minimax-m2.5:free
4. poolside/laguna-m.1:free
5. qwen/qwen3-coder:free
```

For **most free sustained API volume**, use:

```text
1. Cerebras gpt-oss-120b
2. Cerebras qwen-3-235b-a22b-instruct-2507 until deprecation
3. Groq qwen/qwen3-32b
4. Groq openai/gpt-oss-120b
```

For **fast fallback coding loops**, use:

```text
1. Groq qwen/qwen3-32b
2. Groq openai/gpt-oss-120b
3. Cerebras gpt-oss-120b
```

Do not send private repo secrets, API keys, customer data, or proprietary source code into free tiers without reviewing each provider’s data-use terms. For example, Google’s free Gemini tier notes that submitted content may be used to improve products. ([Google AI for Developers][6])
