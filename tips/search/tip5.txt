Fantastic brief. Here’s the strongest stack I’d build **as of May 11, 2026** for agents that need safe, privacy-conscious internet search, publication search, and durable knowledge management.

## Bottom line: the best agent search stack

I would **not** use one tool. I would build a layered retrieval system:

1. **Privacy-first discovery:** Brave Search API + self-hosted SearXNG
2. **Agent-native web search:** Parallel, Tavily, Exa, Perplexity Sonar/Search, or You.com, depending on whether you want raw results, extracted content, or synthesized answers
3. **Publications:** OpenAlex + Semantic Scholar + Crossref + Unpaywall as the core, with PubMed/Europe PMC/arXiv/CORE/Lens/Dimensions/Elicit/scite/Consensus as specialists
4. **Communities and code:** GitHub official APIs, Stack Exchange API, Hacker News/Algolia, Bluesky/ATProto; use Reddit/X/LinkedIn only through approved official APIs
5. **Extraction:** Jina Reader, Firecrawl, Crawl4AI, Trafilatura, Readability
6. **Knowledge management:** Qdrant or LanceDB, with strict provenance, hashes, retrieval timestamps, licenses, and citation metadata

My highest-confidence default is: **Brave Search API + self-hosted SearXNG + OpenAlex/Semantic Scholar/Crossref/Unpaywall + Firecrawl/Jina/Crawl4AI + Qdrant.**

## 1. Best safe/private web search tools for agents

### 1. Brave Search API — best default commercial search API

Brave is my top default for production agents that need privacy-conscious web search. It has an official JSON API, its own independent web index, documented endpoint examples, and an agent/MCP ecosystem. Brave says its API uses an independent index rather than repackaging Google/Bing, and its docs show the `X-Subscription-Token` API pattern for `/res/v1/web/search`. ([Brave][1])

Why I like it:

| Dimension    | Assessment                                                                                                                                                                                                                                                                                   |
| ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Privacy      | Strong. Brave says API query records tied to a customer account are retained for up to 90 days for billing/troubleshooting, and says it does not collect identifiers that link a query to an individual or device; Brave also announced enterprise Zero Data Retention options. ([Brave][2]) |
| Agent fit    | Excellent. Official MCP server and agent skills exist, including web, news, image, video, local/place search, LLM context, and summarization support. ([GitHub][3])                                                                                                                          |
| Independence | Excellent. Brave positions the API as its own independent index, not a Google/Bing scraper. ([Brave][1])                                                                                                                                                                                     |
| Caveat       | Brave search results do **not** grant copyright/storage rights to third-party webpage content unless your plan explicitly permits storage. ([Brave][1])                                                                                                                                      |

**Recommendation:** Make Brave the primary web search backend for general web grounding, company research, public technical research, and current-events discovery.

### 2. SearXNG — best self-hosted/private metasearch layer

SearXNG is the best self-hosted privacy-preserving metasearch option. It aggregates results from multiple search services/databases and says users are neither tracked nor profiled. Its API can return JSON/CSV/RSS via the `format` parameter, though JSON often must be enabled in your instance config. ([GitHub][4])

Why I like it:

| Dimension    | Assessment                                                                                                                                                                                                                                              |
| ------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Privacy      | Very strong when self-hosted. SearXNG private-instance docs describe removing cookies, using random browser profiles per request, presenting the instance IP instead of the user IP, and optional proxy/Tor configuration. ([SearXNG Documentation][5]) |
| Agent fit    | Good. JSON output makes it usable for agents.                                                                                                                                                                                                           |
| Independence | Medium. It is a metasearch engine, not its own full web index.                                                                                                                                                                                          |
| Caveat       | Do **not** abuse public SearXNG instances. Host your own and enable JSON explicitly. ([OpenClaw][6])                                                                                                                                                    |

**Recommendation:** Use SearXNG as a **privacy broker/fallback** and as a way to compare results across engines without sending user-identifying data to every upstream.

### 3. DuckDuckGo and Startpage — good human privacy search, weak agent API story

DuckDuckGo is excellent as a human privacy search engine; it says it does not track searches or browsing history. But its official Instant Answer API is not a full web search results API and “does not include all links,” which makes it weak as a primary agent search backend. ([DuckDuckGo][7])

Startpage is also strong for human private search and says it does not save or sell search history, but I would not make it a core agent tool because it does not provide a broadly documented, production-oriented search-results API comparable to Brave, Tavily, Exa, or Parallel. ([Startpage][8])

**Recommendation:** Good for humans. For agents, avoid scraping them; use only documented APIs.

### 4. Parallel — strongest emerging agent-native search candidate

Parallel is very interesting for agent search. It positions itself as open-web infrastructure for AI, says it is SOC 2 Type II certified, has Search/Extract APIs, and offers an MCP server. In April 2026 it announced free default access for agents through MCP, with dense excerpts, markdown, and an index of billions of pages. ([Parallel Web][9])

Parallel also publicly compares itself on enterprise controls and says it offers zero data retention and contractual non-training commitments. ([Parallel Web][10])

**Recommendation:** Evaluate Parallel next to Brave. Use it when your agents need **token-efficient excerpts** rather than classic search-result snippets.

### 5. Tavily — best “agent/RAG search workflow” API

Tavily is purpose-built for AI agents and RAG. Its docs and product pages emphasize real-time search, extraction, crawl, research workflows, filtering, and built-in security/privacy/content-validation layers to block PII leakage, prompt injection, and malicious sources. ([Tavily][11])

Tavily also offers search, extract, crawl, and research modes; its Research API can run multiple searches, analyze sources, and produce a report. ([Tavily Docs][12])

**Recommendation:** Use Tavily for **agentic research workflows**, especially when you want fewer orchestration steps.

Exa is excellent for semantic discovery, company research, coding-agent research, and “find pages like this” workflows. Its Contents API returns clean structured content from URLs, including complex pages and PDFs, and Exa has a Code/Context API designed for coding agents across GitHub repos, docs, Stack Overflow, and more. Perplexity’s API platform includes Search, Sonar, Agent, and Embeddings APIs. Its Sonar models are web-grounded and return citations; Perplexity’s docs state Sonar API has a strict Zero Data Retention policy and does not train on customer data. ([Perplexity][15])

## 4. GitHub, X, LinkedIn, Reddit, and community sources

### GitHub

Use the official REST/GraphQL APIs and code search. GitHub’s REST docs show authenticated requests get higher limits; GraphQL has point-based limits; search endpoints have separate custom limits. ([GitHub Docs][32])

**Best use:** code examples, repo discovery, issue/PR knowledge, API usage patterns, library health.

### Reddit

Use the official Reddit Data API only. Reddit’s 2026 Responsible Builder Policy says approval is required before accessing Reddit data via API, and the Data API requires OAuth; Reddit documents free eligible limits around 100 QPM per OAuth client ID. ([Reddit Help][33])

**Best use:** public sentiment and troubleshooting, not durable training data unless rights and terms are clear.

Use official X API endpoints. X’s Search Posts APIs support recent search over the last seven days and full-archive search back to 2006, but full archive access is pay-per-use/Enterprise. ([X Developer Platform][34])

**Best use:** current public discourse and real-time signals, only when cost and terms justify it.

Do not scrape LinkedIn. LinkedIn API access is OAuth-based, most permissions and partner programs require explicit approval, and LinkedIn help says third-party crawlers/bots/extensions that scrape or automate LinkedIn are not allowed. ([Microsoft Learn][35])

**Best use:** official approved APIs only, especially organization/page/member content you are authorized to access.

These are often more agent-friendly than closed social platforms. Stack Exchange has a documented JSON API with search endpoints and throttles; Hacker News has an official API plus Algolia search; Bluesky/AT Protocol has a public firehose and open protocol concepts for public conversation. ## 6. Knowledge management: where agents should store what they learn

### Qdrant — best default vector DB for Rust-heavy agent systems

Qdrant is open-source, written in Rust, and supports vector search, hybrid retrieval, metadata filtering, and JSON payloads. Its docs show collection creation, upserting points, payloads, and universal query endpoints. ([Qdrant][43])

LanceDB stores multimodal data, metadata, embeddings, vector search, full-text search, and SQL-style querying in the same table; it also has a Rust crate. ([LanceDB][44])

Use **Qdrant** for service-based production systems. Use **LanceDB** for local-first or embedded agent memory.

## 7. Safety architecture for search agents

Every webpage, PDF, Reddit comment, README, or article must be treated as **untrusted input**, not instructions. OWASP’s LLM Top 10 includes prompt injection, insecure output handling, training data poisoning, model denial of service, and supply-chain risk; OWASP defines prompt injection as manipulating model responses through inputs to alter behavior. ([OWASP][46])

Meta’s “Agents Rule of Two” is a useful design rule: avoid combining all three of these in one agent session—untrusted data, private data, and external communication/action. ([Meta AI][47])

My hard rules:

1. **Search/extract agents should not have write/action permissions.**
2. **Fetched web content is data, never instructions.**
3. **Separate browsing from private-memory access.**
4. **Respect robots.txt and site terms.** RFC 9309 standardizes the Robots Exclusion Protocol, though robots.txt is not access authorization. ([RFC Editor][48])
5. **Cache responsibly.** Respect provider storage/copyright limits.
6. **Require citations for durable knowledge.**
7. **Store hashes and retrieval timestamps.**
8. **Use rate limits and backoff.**
9. **Do not scrape LinkedIn, Reddit, X, or paywalled publications without explicit authorization.**
10. **Run search tools in a low-privilege sandbox with no secrets except that provider’s API key.**

## 1. Brave Search API client

```rust
use anyhow::{Context, Result};
use reqwest::Client;
use serde::{Deserialize, Serialize};

#[derive(Debug, Clone, Serialize)]
pub struct SearchHit {
    pub title: String,
    pub url: String,
    pub description: Option<String>,
    pub provider: String,
}

#[derive(Debug, Deserialize)]
struct BraveResponse {
    web: Option<BraveWeb>,
}

#[derive(Debug, Deserialize)]
struct BraveWeb {
    #[serde(default)]
    results: Vec<BraveResult>,
}

#[derive(Debug, Deserialize)]
struct BraveResult {
    title: String,
    url: String,
    description: Option<String>,
}

pub async fn brave_search(client: &Client, query: &str, count: u8) -> Result<Vec<SearchHit>> {
    let api_key = std::env::var("BRAVE_SEARCH_API_KEY")
        .context("BRAVE_SEARCH_API_KEY is not set")?;

    let response = client
        .get("https://api.search.brave.com/res/v1/web/search")
        .header("X-Subscription-Token", api_key)
        .header("Accept", "application/json")
        .query(&[
            ("q", query.to_string()),
            ("count", count.min(20).to_string()),
            ("country", "us".to_string()),
            ("search_lang", "en".to_string()),
            ("extra_snippets", "true".to_string()),
        ])
        .send()
        .await?
        .error_for_status()?
        .json::<BraveResponse>()
        .await?;

    Ok(response
        .web
        .map(|w| w.results)
        .unwrap_or_default()
        .into_iter()
        .map(|r| SearchHit {
            title: r.title,
            url: r.url,
            description: r.description,
            provider: "brave".to_string(),
        })
        .collect())
}
```

```rust
use anyhow::{Context, Result};
use reqwest::Client;
use serde_json::{json, Value};

pub async fn jina_reader_markdown(client: &Client, url: &str) -> Result<String> {
    let endpoint = format!("https://r.jina.ai/{url}");

    let mut request = client
        .get(endpoint)
        .header("Accept", "text/plain")
        .header("User-Agent", "safe-agent-reader/0.1");

    if let Ok(key) = std::env::var("JINA_API_KEY") {

    }

    Ok(request
        .send()
        .await?
        .error_for_status()?
        .text()
        .await?)
}

pub async fn firecrawl_scrape_markdown(client: &Client, url: &str) -> Result<String> {
    let api_key = std::env::var("FIRECRAWL_API_KEY")
        .context("FIRECRAWL_API_KEY is not set")?;

    let value = client
        .post("https://api.firecrawl.dev/v1/scrape")
        .bearer_auth(api_key)
        .json(&json!({
            "url": url,
            "formats": ["markdown"],
            "onlyMainContent": true
        }))
        .send()
        .await?
        .error_for_status()?
        .json::<Value>()
        .await?;

    let markdown = value
        .pointer("/data/markdown")
        .and_then(|v| v.as_str())
        .unwrap_or_default()
        .to_string();

    Ok(markdown)
}
```

## 5. Qdrant knowledge upsert with provenance

```rust
use anyhow::Result;
use reqwest::{Client, StatusCode};
use serde_json::{json, Value};
use sha2::{Digest, Sha256};
use uuid::Uuid;

fn sha256_hex(input: &str) -> String {
    let mut hasher = Sha256::new();
    hasher.update(input.as_bytes());
    hex::encode(hasher.finalize())
}

pub async fn ensure_qdrant_collection(
    client: &Client,
    qdrant_base: &str,
    collection: &str,
    vector_size: usize,
) -> Result<()> {
    let endpoint = format!(
        "{}/collections/{}",
        qdrant_base.trim_end_matches('/'),
        collection
    );

    let response = client
        .put(endpoint)
        .json(&json!({
            "vectors": {
                "size": vector_size,
                "distance": "Cosine"
            }
        }))
        .send()
        .await?;

    if response.status().is_success() || response.status() == StatusCode::CONFLICT {
        Ok(())
    } else {
        response.error_for_status()?;
        Ok(())
    }
}

pub async fn upsert_knowledge_chunk(
    client: &Client,
    qdrant_base: &str,
    collection: &str,
    source_url: &str,
    title: &str,
    provider: &str,
    source_type: &str,
    retrieved_at_iso8601: &str,
    chunk_text: &str,
    embedding: Vec<f32>,
) -> Result<()> {
    let point_id = Uuid::new_v5(&Uuid::NAMESPACE_URL, source_url.as_bytes()).to_string();
    let content_hash = sha256_hex(chunk_text);

    let payload: Value = json!({
        "source_url": source_url,
        "title": title,
        "provider": provider,
        "source_type": source_type,
        "retrieved_at": retrieved_at_iso8601,
        "content_sha256": content_hash,
        "text": chunk_text
    });

    let endpoint = format!(
        "{}/collections/{}/points?wait=true",
        qdrant_base.trim_end_matches('/'),
        collection
    );

    client
        .put(endpoint)
        .json(&json!({
            "points": [
                {
                    "id": point_id,
                    "vector": embedding,
                    "payload": payload
                }
            ]
        }))
        .send()
        .await?
        .error_for_status()?;

    Ok(())
}
```

```rust
pub enum QueryClass {
    Academic,
    News,
    Code,
    Social,
    GeneralWeb,
}

pub fn classify_query(query: &str) -> QueryClass {
    let q = query.to_lowercase();

    if q.contains("doi:")
        || q.contains("pubmed")
        || q.contains("pmid")
        || q.contains("arxiv")
        || q.contains("systematic review")
        || q.contains("citation")
        || q.contains("paper")
    {
        QueryClass::Academic
    } else if q.contains("github")
        || q.contains("crate")
        || q.contains("repository")
        || q.contains("api example")
        || q.contains("stack overflow")
    {
        QueryClass::Code
    } else if q.contains("today")
        || q.contains("latest")
        || q.contains("breaking")
        || q.contains("news")
        || q.contains("recent")
    {
        QueryClass::News
    } else if q.contains("reddit")
        || q.contains("x.com")
        || q.contains("twitter")
        || q.contains("linkedin")
        || q.contains("bluesky")
    {
        QueryClass::Social
    } else {
        QueryClass::GeneralWeb
    }
}
```

Routing policy:

```text
Academic:
  OpenAlex → Semantic Scholar → Crossref → Unpaywall → domain-specific APIs

News:
  GDELT / Media Cloud → Brave News / Tavily / You.com / Event Registry

Code:
  GitHub REST/GraphQL/code search → Exa code/context → Stack Exchange → HN

Social:
  Official API only: Reddit, X, LinkedIn; prefer Bluesky/HN/Stack Exchange when possible

General web:
  Brave Search → SearXNG → Parallel/Tavily/Exa → extraction layer → Qdrant/LanceDB
```

## My “best possible” final tool bundle

For a serious agent platform, I would ship this:

```text
Primary web search:
  Brave Search API

Private fallback/meta-search:
  Self-hosted SearXNG

Agent-native search:
  Parallel + Tavily + Exa
  Optional: Perplexity Sonar or You.com Research for synthesized cited answers

Academic:
  OpenAlex
  Semantic Scholar
  Crossref
  Unpaywall
  PubMed / Europe PMC / arXiv / CORE
  Optional: Lens, Dimensions, Elicit, scite, Consensus

News/publications:
  GDELT
  Media Cloud
  NewsAPI or Event Registry / NewsAPI.ai

Code/community:
  GitHub REST + GraphQL + Code Search
  Stack Exchange API
  Hacker News official API + Algolia
  Bluesky/AT Protocol firehose

Restricted social:
  Reddit official API with approval
  X official API
  LinkedIn approved APIs only

Extraction:
  Jina Reader
  Firecrawl
  Crawl4AI
  Trafilatura
  Readability

Knowledge:
  Qdrant for production
  LanceDB for embedded/local
  PostgreSQL/SQLite for audit log and crawl frontier
  Object storage for raw snapshots
```

The most important design choice: **keep search, extraction, and memory separate**. Search tools find candidates, extraction tools create clean text, verification tools validate sources, and the memory layer stores only provenance-rich, citation-ready knowledge.
