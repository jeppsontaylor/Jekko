As of **May 2026**, my answer is: the most human-like design I found is **Hindsight**, but the best practical architecture would be a **hybrid**: **Hindsight-style cognitive networks + Mastra Observational Memory + Graphiti/Zep temporal graph + HippoRAG associative retrieval + Voyager/Memp procedural skills**.

I’m using “human-like” as an engineering analogy: separate working, episodic, semantic, procedural memory; reflection; temporal grounding; decay or belief update; fast associative recall; and experience-to-skill compounding. CoALA is the clean cognitive-science blueprint for this framing: it organizes language agents around modular memory, internal/external actions, and decision loops. LongMemEval and LoCoMo are the most useful public benchmarks I found for long-term conversational memory, but benchmark claims are not perfectly comparable because systems use different judges, models, prompts, and retrieval settings. ([Hugging Face][1])

## 1. Hindsight — best overall human-like memory

**Why it ranks first:** Hindsight is the most explicitly “cognitive” recent architecture I found. It treats memory as a first-class reasoning substrate instead of a vector-store attachment. It organizes memory into distinct logical networks for **world facts**, **agent experiences**, **observations/entity summaries**, and **evolving beliefs/opinions**, then operates through **retain → recall → reflect**. That distinction between evidence, experience, and belief is exactly what most agent memories lack. The paper reports strong results on LongMemEval and LoCoMo, including 83.6% with an open 20B model and 91.4% when scaled further; the GitHub repo describes biomimetic memory banks for facts, experiences, and mental models. ([Hugging Face][2])

**Build schematic:**

```text
conversation / tool trace / observation
        ↓ retain
extract entities, facts, events, time, source, confidence
        ↓
World Network      = stable external facts
Experience Network = what the agent/user did or observed
Entity Summaries   = synthesized profiles over time
Belief Network     = evolving opinions, preferences, hypotheses
        ↓ recall
hybrid search + temporal/entity traversal
        ↓ reflect
update beliefs, resolve conflicts, explain reasoning path
```

**Use this when:** the agent must become a better research partner over months, not merely remember snippets. For OpenQG-style science work, I would use Hindsight’s separation between **world knowledge**, **research episodes**, **entity/concept summaries**, and **belief/hypothesis state**.

---

## 2. Mastra Observational Memory — best “subconscious” memory

**Why it ranks second:** Observational Memory is the most elegant recent “human-like but fast” design. It uses two background agents: an **Observer** that compresses raw conversation into dense dated observations, and a **Reflector** that periodically restructures and condenses those observations. The main agent does not dynamically query memory every turn; it sees a stable memory prefix plus recent conversation. Mastra reports 94.87% on LongMemEval with GPT-5-mini, 84.23% with GPT-4o, and emphasizes stable, cacheable context rather than per-turn vector retrieval. ([Mastra][3])

**Build schematic:**

```text
raw message stream
        ↓ token threshold
Observer agent
        ↓
dated observation log:
  - what happened
  - who/what entities changed
  - priority
  - explicit date / referenced date / relative date
        ↓ size threshold
Reflector agent
        ↓
compressed, reorganized observation memory
        ↓
stable prompt prefix for Actor agent
```

**Use this when:** you want the fastest human-feeling continuity. It mimics the way people do not remember every word, but keep a compressed temporal sense of “what has been happening.” For coding or research agents, it is excellent as a **rolling autobiographical layer**.

---

## 3. MIRIX — best multimodal multi-agent memory

**Why it ranks third:** MIRIX is one of the broadest memory systems: it has **six memory types** — core, episodic, semantic, procedural, resource, and knowledge vault — and an **eight-agent architecture** that routes inputs to specialized memory managers. It also supports multimodal data such as screenshots, not just text, and the paper reports 85.4% on LoCoMo plus large gains on ScreenshotVQA while reducing storage. The docs show a meta-agent routing content into six memory managers backed by a shared database, with BM25 and embedding retrieval. ([Hugging Face][4])

**Build schematic:**

```text
input: chat, screen, doc, code, voice
        ↓
Meta Agent classifies content
        ↓
Core Memory Manager        = identity, durable profile
Episodic Memory Manager    = events and activities
Semantic Memory Manager    = facts and concepts
Procedural Memory Manager  = workflows and habits
Resource Memory Manager    = documents and external artifacts
Knowledge Vault Manager    = sensitive or protected knowledge
        ↓
shared local DB
        ↓
BM25 + vector + field-specific retrieval
```

**Use this when:** you want a serious personal or organizational AI memory that can handle **text + visuals + workflows + resources**. For OpenQG, I would borrow the six-type memory taxonomy, but I would avoid storing sensitive vault content in the research repo.

---

## 4. Zep / Graphiti — best temporal knowledge graph memory

**Why it ranks fourth:** Graphiti is a temporal context-graph engine: it tracks how facts change over time, maintains provenance to source data, and supports learned or prescribed ontology. Zep’s paper and blog describe Graphiti as the core component behind a temporal knowledge-graph memory architecture for agents, reporting better DMR performance than MemGPT and latency/token reductions in LongMemEval-style settings. ([GitHub][5])

**Build schematic:**

```text
episode: conversation, paper, code run, dataset event
        ↓ extract
entities, relations, facts, timestamps, provenance
        ↓ upsert
temporal graph:
  node: entity/concept/paper/person/dataset
  edge: supports, contradicts, cites, updates, depends_on
  valid_from / valid_to / observed_at
        ↓ query
semantic search + keyword search + graph traversal + temporal filters
        ↓
context graph slice for agent
```

**Use this when:** the memory must reason over **changing truth**: “this was true in January, superseded in March, contradicted by paper X.” For scientific knowledge compounding, this is essential. A flat vector store cannot reliably handle contradictions, provenance, and temporal validity.

---

## 5. HippoRAG / HippoRAG 2 — best hippocampal associative retrieval

**Why it ranks fifth:** HippoRAG is explicitly inspired by hippocampal indexing theory. It combines LLM extraction, knowledge graphs, and Personalized PageRank to solve multi-hop/path-finding retrieval. The paper reports up to 20% improvement on multi-hop QA and says single-step HippoRAG can be 10–30× cheaper and 6–13× faster than iterative retrieval such as IRCoT. HippoRAG 2 extends this toward memory, emphasizing associativity and sense-making. ([Hugging Face][6])

**Build schematic:**

```text
documents / papers / notes
        ↓ OpenIE / entity extraction
triples and entity graph
        ↓
query → detect seed entities
        ↓
Personalized PageRank over graph
        ↓
retrieve top connected passages / claims
        ↓
LLM reader answers with evidence
```

**Use this when:** you have a large scientific corpus and need to answer questions that require chains of facts across papers. For OpenQG, this is one of the best designs for “show me the hidden bridge between theory A, observable B, dataset C, and method D.”

---

## 6. A-MEM — best self-organizing concept memory

**Why it ranks sixth:** A-MEM is a Zettelkasten-inspired agentic memory system. New memories are converted into structured notes with context, keywords, tags, and links; the system analyzes historical memories, creates meaningful connections, and evolves older memories as new ones arrive. Its GitHub repo describes dynamic indexing, linking, memory evolution, and agent-driven memory organization using ChromaDB. ([Hugging Face][7])

**Build schematic:**

```text
new experience / note
        ↓
LLM generates:
  summary
  context
  keywords
  tags
  category
  timestamp
        ↓
semantic search old notes
        ↓
create links:
  similar_to, extends, contradicts, refines, prerequisite_of
        ↓
memory evolution:
  update old note context/tags/links
        ↓
agentic search over linked notes
```

**Use this when:** you want memory that **self-organizes concepts** instead of just retrieving chunks. For a research memory bank, A-MEM is especially good for building living concept maps and literature trails.

---

## 7. MemGPT / Letta — best self-editing memory hierarchy

**Why it ranks seventh:** MemGPT introduced the “LLM as operating system” memory analogy: the model manages a virtual context using fast in-context memory and slower external memory, similar to paging. Letta continues that architecture with memory blocks, recall memory, archival memory, and tools that let agents insert, replace, rethink, and search memory. ([MemGPT][8])

**Build schematic:**

```text
context window:
  system instructions
  core memory blocks
  recent messages
        ↕ tools
recall memory:
  searchable conversation history
        ↕ tools
archival memory:
  long-term semantic store
        ↕
agent decides:
  search memory
  insert memory
  replace memory
  rethink memory block
  compact old context
```

**Use this when:** you want an agent that can actively manage its own memory. It is less human-like than Hindsight or MIRIX, but it is foundational: the agent can **choose what to page into context**, not just receive whatever RAG retrieves.

---

## 8. Mem0 / Mem0-Graph — best production memory API pattern

**Why it ranks eighth:** Mem0 is a practical, scalable memory layer for assistants and agents. It emphasizes user/session/agent state, adaptive personalization, extraction, consolidation, retrieval, and a graph-based variant for relationships. The GitHub repo reports research highlights including LoCoMo, LongMemEval, and BEAM results, and the paper frames Mem0 as a production-ready long-term memory architecture for multi-session coherence. ([GitHub][9])

**Build schematic:**

```text
conversation turn
        ↓
salience extractor
        ↓
memory operation:
  add new fact
  update old fact
  delete expired fact
  merge duplicate
  link entity relationship
        ↓
user/session/agent memory stores
        ↓
graph + vector + metadata retrieval
        ↓
personalized context injection
```

**Use this when:** you need to ship an AI product with memory quickly. It may not be the most cognitively rich architecture, but it is strong for scalable personalization and clean API integration.

---

## 9. Voyager + Memp — best procedural skill memory

**Why it ranks ninth:** Human intelligence is not just facts; it is reusable skill. Voyager showed a powerful procedural-memory pattern: the agent explores, writes executable code skills, stores them in an ever-growing skill library, retrieves relevant skills later, and composes them into more complex behaviors. Memp pushes this idea into a more general agent procedural memory: it distills trajectories into fine-grained instructions and higher-level scripts, then continuously updates, corrects, and deprecates them. ([GitHub][10])

**Build schematic:**

```text
attempt task
        ↓
observe success/failure/tool errors
        ↓
self-critique and repair
        ↓
if success:
  distill into skill:
    name
    trigger
    preconditions
    code or procedure
    tests
    embeddings
        ↓
retrieve top skills for new task
        ↓
compose / adapt / execute
```

**Use this when:** the agent must get better at doing, not just remembering. For OpenQG, this is how the system should learn reusable scripts: “download this dataset,” “parse this arXiv PDF,” “fit this likelihood,” “generate this benchmark manifest.”

---

## 10. Memvid Smart-Frame Memory — fastest portable recall design

**Why it ranks tenth:** Memvid is less human-like than the top systems, but it is one of the most interesting speed/portability designs. It packages data, embeddings, indices, metadata, WAL, lexical index, vector index, and time index into a single `.mv2` file. Its repo claims extremely low-latency retrieval and describes append-only “Smart Frames” with timestamps, checksums, compression, time-travel debugging, and single-file portability. Treat its benchmark claims cautiously because some benchmark comparisons are vendor-reported, but the design is very strong for local-first agent memory capsules. ([GitHub][11])

**Build schematic:**

```text
memory append
        ↓
Smart Frame:
  content
  timestamp
  checksum
  metadata
  embedding
        ↓
single file:
  header
  WAL
  compressed data segments
  Tantivy lexical index
  HNSW vector index
  time index
  footer / table of contents
        ↓
local hybrid recall
```

**Use this when:** you want memory to be **portable, versioned, offline, and fast**. I would not use it as the only “brain,” but I would consider it as a fast local substrate for memory capsules.

## Foundational designs still worth stealing from

**Generative Agents** introduced a very human-readable loop: memory stream, recency/relevance/importance retrieval, reflection, planning, and action. It remains one of the clearest examples of “believable human behavior” from memory, even if newer systems are stronger on benchmarks. ([Hugging Face][12])

**MemoryBank** is worth stealing from because it explicitly uses an Ebbinghaus-inspired forgetting/reinforcement mechanism for long-term companion memory and user personality adaptation. ([AAAI Publications][13])

**Reflexion** is the classic lightweight “learn from failure” loop: actor, evaluator, self-reflection, and episodic reflection memory. It is not a full long-term memory architecture, but it is extremely useful as a per-task improvement layer. ([Princeton University][14])

## My build recommendation

For a serious compounding-knowledge agent, I would not choose one design. I would compose them like this:

```text
1. Working/Core memory
   → MemGPT/Letta-style pinned context blocks

2. Autobiographical rolling memory
   → Mastra Observational Memory observer/reflector log

3. Structured cognitive memory
   → Hindsight world/experience/entity/belief networks

4. Temporal relational memory
   → Graphiti/Zep temporal knowledge graph

5. Associative scientific recall
   → HippoRAG graph + Personalized PageRank retrieval

6. Self-organizing notes
   → A-MEM/Zettelkasten concept-link evolution

7. Procedural skill memory
   → Voyager/Memp executable skill and workflow library

8. Fast local memory substrate
   → Memvid/SQLite/Tantivy/Qdrant-style hybrid indexes

9. Canonical source of truth
   → git-backed Markdown/YAML/JSONL manifests

10. Evaluation gate
   → LongMemEval/LoCoMo-style tests plus project-specific recall, contradiction, and provenance tests
```

The winning principle is:

```text
Do not make the vector DB the memory.
Make structured, source-controlled memory the truth.
Use graphs, vectors, observations, and skills as compiled indexes over it.
```

For your OpenQG goal, the best “human-like” memory would be:

```text
Hindsight for epistemic structure
+ Graphiti for time/provenance
+ HippoRAG for scientific association
+ A-MEM for self-organizing concept notes
+ Voyager/Memp for reusable research/code skills
+ Observational Memory for low-latency continuity
```

That combination gives you fast recall, self-organization, experience compounding, concept formation, procedural learning, and a way to keep the agent from overwhelming its context window.
