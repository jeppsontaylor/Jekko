I’m interpreting “connections/hour” as **API requests/hour**. Most providers publish **RPM/RPD/TPM**, not TCP connection counts, so below I show either the official **RPH** or the calculated burst equivalent **RPM × 60**, plus the daily cap that often becomes the real throttle.

## Best answer: sign up for these first

For **free coding API access today**, the best practical stack is:

1. **Gemini API** for strongest free frontier coding model access.
2. **Cloudflare Workers AI** for the best free RPM, but limited by daily “neurons.”
3. **Cerebras** for strong free hourly request limits and very fast inference.
4. **Groq** for speed, but stricter token/day caps.
5. **OpenRouter** for the broadest set of powerful free models, but only **50 free-model requests/day** on a pure free account.
6. **GitHub Models** for limited free access to GPT-4.1 and other models via a GitHub token.

Gemini and Mistral do **not** publish stable global free-tier RPM/RPH tables; they tell you to check the active limits in their dashboards. Google’s Gemini docs say free Gemini API limits are enforced across RPM/TPM/RPD and exact limits are visible in AI Studio, while Gemini 3 Flash and Gemini 3.1 Flash-Lite have API free tiers; Gemini 3.1 Pro does **not** have a free API tier. ([Google AI for Developers][1])

## One Python client for most of the top 20

This uses OpenAI-compatible chat completions for Gemini, OpenRouter, Cloudflare, Cerebras, Groq, and GitHub.

```python
import os
from openai import OpenAI

PROVIDERS = {
    "gemini": {
        "base_url": "https://generativelanguage.googleapis.com/v1beta/openai/",
        "api_key": os.environ["GEMINI_API_KEY"],
    },
    "openrouter": {
        "base_url": "https://openrouter.ai/api/v1",
        "api_key": os.environ["OPENROUTER_API_KEY"],
    },
    "cloudflare": {
        "base_url": f"https://api.cloudflare.com/client/v4/accounts/{os.environ['CLOUDFLARE_ACCOUNT_ID']}/ai/v1",
        "api_key": os.environ["CLOUDFLARE_API_TOKEN"],
    },
    "cerebras": {
        "base_url": "https://api.cerebras.ai/v1",
        "api_key": os.environ["CEREBRAS_API_KEY"],
    },
    "groq": {
        "base_url": "https://api.groq.com/openai/v1",
        "api_key": os.environ["GROQ_API_KEY"],
    },
    "github": {
        "base_url": "https://models.github.ai/inference",
        "api_key": os.environ["GITHUB_TOKEN"],
    },
}

def ask(provider_name: str, model: str, prompt: str) -> str:
    cfg = PROVIDERS[provider_name]
    client = OpenAI(api_key=cfg["api_key"], base_url=cfg["base_url"])

    response = client.chat.completions.create(
        model=model,
        messages=[
            {
                "role": "system",
                "content": "You are an expert software engineer. Be concise, correct, and practical.",
            },
            {"role": "user", "content": prompt},
        ],
        temperature=0.2,
    )
    return response.choices[0].message.content

PROMPT = """
Review this Python function for bugs, edge cases, and performance issues.
Return a corrected version with a brief explanation.

def binary_search(xs, target):
    lo, hi = 0, len(xs)
    while lo < hi:
        mid = (lo + hi) // 2
        if xs[mid] == target:
            return mid
        elif xs[mid] < target:
            lo = mid
        else:
            hi = mid
    return -1
"""

# Example: choose one model call at a time.
print(ask("gemini", "gemini-3-flash-preview", PROMPT))
```
