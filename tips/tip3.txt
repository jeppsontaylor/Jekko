## Scope

I treated ÔÇťfreeÔÇŁ as **zero-dollar hosted API access you can call from code**, including free tiers and free monthly credits. I treated ÔÇťconnections/hourÔÇŁ as **requests/hour**. When a provider publishes only RPM, I compute burst RPH as `RPM ├Ś 60`, but the **daily cap, token cap, credits, or concurrency cap may stop you earlier**.

Free limits are volatile. Google explicitly says limits vary by model/tier and should be checked in AI Studio; Groq and several others also expose live limits in dashboards. ([Google AI for Developers][1])

# Setup and example API calls

## 1. OpenRouter

Setup URL: `https://openrouter.ai/keys`

Steps:

1. Create/sign in to OpenRouter.
2. Open **Keys**.
3. Create an API key.
4. Use `:free` model IDs to avoid paid routes.

```bash
export OPENROUTER_API_KEY="your_key_here"

curl https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen/qwen3-coder:free",
    "messages": [
      {
        "role": "user",
        "content": "Implement a Rust LRU cache with unit tests and explain edge cases."
      }
    ]
  }'
```

Swap in:

```text
nvidia/nemotron-3-super-120b-a12b:free
nvidia/nemotron-3-nano-30b-a3b:free
openrouter/free
```

## Practical recommendation

For a free coding stack, I would use this order:

1. **Hardest coding / repo reasoning:** `qwen/qwen3-coder:free` on OpenRouter, then `gemini-2.5-pro`.
2. **Daily coding throughput:** Groq `qwen/qwen3-32b` and `llama-3.3-70b-versatile`.
3. **Fast large-model bursts:** Cerebras `gpt-oss-120b` or Qwen 235B while available.
4. **Edge/app experiments:** Cloudflare Workers AI.
5. **Fallbacks:** Mistral Codestral, GitHub Models, Hugging Face, NVIDIA NIM.

Do not create duplicate accounts or rotate keys to evade caps; use these as legitimate free-tier options and upgrade the single provider that proves most useful.
