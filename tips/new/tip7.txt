## Best free hosted coding LLM APIs I found — ranked for power first, then usable free bandwidth

As of **May 6, 2026**, the most important caveat is that **OpenRouter has several of the strongest free coding models, but its free quota is small**: `:free` models are capped at **20 RPM**, with **50 requests/day** unless you have purchased at least 10 credits, which raises the daily cap to **1,000 requests/day**. Creating extra keys/accounts does not raise the limit. ([OpenRouter][1])

**How I’m calculating RPH:** when a provider publishes only RPM, I list “burst RPH” as `RPM × 60`, but daily/monthly caps still dominate. For example, **Groq `qwen/qwen3-32b` is 60 RPM = 3,600 burst RPH, but only 1,000 requests/day on the free plan**, so you cannot sustain 3,600/hour all day. Groq’s docs define RPM/RPD/TPM/TPD and state limits apply at the organization level. ([GroqCloud][2])

## My practical recommendation

For **best coding power**, start with **OpenRouter Qwen3 Coder**, **MiniMax M2.5**, **Poolside Laguna M.1**, and **Gemini 2.5 Pro**. For **best free throughput with published limits**, use **Cerebras GPT-OSS-120B**, **Cerebras Qwen3-235B until May 27**, **Groq Qwen3-32B**, and **Cloudflare Qwen2.5-Coder**.

For a real coding agent, I would configure fallbacks in this order:

`qwen/qwen3-coder:free` → `minimax/minimax-m2.5:free` → `poolside/laguna-m.1:free` → `gpt-oss-120b` on Cerebras → `qwen/qwen3-32b` on Groq → `@cf/qwen/qwen2.5-coder-32b-instruct` on Cloudflare.

# Setup + example API calls

## 1. OpenRouter — best place for many free top coding models

Setup:

1. Go to [https://openrouter.ai/keys](https://openrouter.ai/keys)
2. Create an API key.
3. Pick a `:free` model from the table.
4. Remember the shared free-model limit: **20 RPM**, **50 RPD**, or **1,000 RPD after ≥10 credits**. ([OpenRouter][1])

```bash
export OPENROUTER_API_KEY="sk-or-..."
export MODEL_ID="qwen/qwen3-coder:free"

curl https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "'"$MODEL_ID"'",
    "messages": [
      {
        "role": "system",
        "content": "You are an expert coding assistant. Be concise and include tests."
      },
      {
        "role": "user",
        "content": "Write a Python function that topologically sorts a DAG and include pytest tests."
      }
    ],
    "temperature": 0.2
  }'
```

Swap `MODEL_ID` for:

```text
tencent/hy3-preview:free
qwen/qwen3-coder:free
inclusionai/ling-2.6-1t:free
minimax/minimax-m2.5:free
poolside/laguna-m.1:free
nvidia/nemotron-3-super-120b-a12b:free
z-ai/glm-4.5-air:free
openai/gpt-oss-120b:free
openai/gpt-oss-20b:free
```

## 2. Cerebras — best free published bandwidth for strong models

Setup:

1. Go to [https://cloud.cerebras.ai/](https://cloud.cerebras.ai/)
2. Open the API Playground and create a free API key. Cerebras explicitly says to get a free key from the API playground and uses OpenAI-compatible base URL `https://api.cerebras.ai/v1`. ([Cerebras Inference][20])
3. Use `gpt-oss-120b` first for coding.

```bash
export CEREBRAS_API_KEY="..."

curl https://api.cerebras.ai/v1/chat/completions \
  -H "Authorization: Bearer $CEREBRAS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-oss-120b",
    "messages": [
      {
        "role": "user",
        "content": "Implement an LRU cache in Python with O(1) get/put and include unit tests."
      }
    ],
    "temperature": 0.2
  }'
```

Best Cerebras model IDs:

```text
gpt-oss-120b
zai-glm-4.7
qwen-3-235b-a22b-instruct-2507
```

export GROQ_API_KEY="..."

## 4. Google Gemini API — strongest free quality, limits shown in AI Studio

Setup:

1. Go to [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey)
2. Create or view a Gemini API key. Google says Gemini keys are created and managed from Google AI Studio’s API Keys page. ([Google AI for Developers][4])
3. Check [https://aistudio.google.com/](https://aistudio.google.com/) for your **active RPM/TPM/RPD**, because Google’s docs say limits vary by model and tier and are viewed in AI Studio. ([Google AI for Developers][22])

```bash
export GEMINI_API_KEY="..."

curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-pro:generateContent?key=$GEMINI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "contents": [
      {
        "parts": [
          {
            "text": "Write a Rust parser for simple arithmetic expressions with unit tests."
          }
        ]
      }
    ]
  }'
```

Good Gemini model IDs:

```text
gemini-2.5-pro
gemini-2.5-flash
gemini-2.5-flash-lite
```

## 5. Cloudflare Workers AI — best free RPM for smaller hosted coding models

Setup:

1. Go to [https://dash.cloudflare.com/?to=/:account/workers-ai](https://dash.cloudflare.com/?to=/:account/workers-ai)
2. Select **Use REST API**.
3. Select **Create a Workers AI API Token**.
4. Copy your API token and Account ID. Cloudflare’s REST guide gives these exact steps. ([Cloudflare Docs][23])

```bash
export CLOUDFLARE_ACCOUNT_ID="..."
export CLOUDFLARE_AUTH_TOKEN="..."

curl "https://api.cloudflare.com/client/v4/accounts/$CLOUDFLARE_ACCOUNT_ID/ai/run/@cf/qwen/qwen2.5-coder-32b-instruct" \
  -X POST \
  -H "Authorization: Bearer $CLOUDFLARE_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {
        "role": "system",
        "content": "You are a senior software engineer."
      },
      {
        "role": "user",
        "content": "Write a Go binary search implementation with table-driven tests."
      }
    ]
  }'
```

Good Cloudflare model IDs:

```text
@cf/qwen/qwen2.5-coder-32b-instruct
@cf/meta/llama-4-scout-17b-16e-instruct
```

## 6. GitHub Models — convenient free API with GitHub PAT, low daily cap

Setup:

1. Browse models at [https://github.com/marketplace/models](https://github.com/marketplace/models)
2. Create a personal access token at [https://github.com/settings/personal-access-tokens](https://github.com/settings/personal-access-tokens)
3. Give it the `models` / `models:read` scope. GitHub’s quickstart says API use requires a GitHub account and PAT with the `models` scope. ([GitHub Docs][12])

```bash
export GITHUB_PAT="github_pat_..."

curl -L -X POST https://models.github.ai/inference/chat/completions \
  -H "Accept: application/vnd.github+json" \
  -H "Authorization: Bearer $GITHUB_PAT" \
  -H "X-GitHub-Api-Version: 2022-11-28" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4.1",
    "messages": [
      {
        "role": "user",
        "content": "Explain this recursive function and rewrite it iteratively in Python."
      }
    ]
  }'
```

## 7. Mistral AI — free Experiment plan, best for Devstral evaluation

Setup:

1. Go to [https://console.mistral.ai/](https://console.mistral.ai/)
2. Activate the **Experiment** plan. Mistral says it is free, requires a verified phone number, and no credit card is required. ([Mistral AI Help Center][24])
3. Create an API key in the console.
4. Check **Admin › Limits** for your exact RPS/TPM/month caps; Mistral publishes the metrics but not a fixed universal free RPM/RPH table. ([Mistral AI][25])

```bash
export MISTRAL_API_KEY="..."

curl https://api.mistral.ai/v1/chat/completions \
  -H "Authorization: Bearer $MISTRAL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "devstral-2512",
    "messages": [
      {
        "role": "user",
        "content": "Patch this failing pytest test and explain the root cause."
      }
    ],
    "temperature": 0.2
  }'
```

The Devstral 2 model card displays `devstral-2512+2`, so verify the exact callable ID with:

```bash
curl https://api.mistral.ai/v1/models \
  -H "Authorization: Bearer $MISTRAL_API_KEY"
```

## 8. Cohere — free trial key, useful reasoning fallback

Setup:

1. Go to [https://dashboard.cohere.com/api-keys](https://dashboard.cohere.com/api-keys)
2. Create an account; Cohere says a trial API key is automatically created for new accounts. ([Cohere][26])
3. Use it for evaluation only; Cohere says trial calls are free but not for production/commercial use. ([Cohere][26])

```bash
export COHERE_API_KEY="..."

curl https://api.cohere.com/v2/chat \
  -H "x-api-key: $COHERE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "command-a-reasoning-08-2025",
    "messages": [
      {
        "role": "user",
        "content": "Find the bug in this algorithm and provide a corrected implementation."
      }
    ]
  }'
```

## 9. Hugging Face Inference Providers — tiny monthly free credits, useful as a router

Hugging Face is not top-ranked for free coding bandwidth because free users get only **$0.10/month** in Inference Provider credits, but it is still useful as a universal router. ([Hugging Face][27])

Setup:

1. Go to [https://huggingface.co/settings/tokens](https://huggingface.co/settings/tokens)
2. Create a fine-grained token with **Make calls to Inference Providers** permission. Hugging Face documents that exact permission. ([Hugging Face][28])

```bash
export HF_TOKEN="hf_..."

curl https://router.huggingface.co/v1/chat/completions \
  -H "Authorization: Bearer $HF_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-oss-120b:fastest",
    "messages": [
      {
        "role": "user",
        "content": "Generate a Python pytest suite for a queue implementation."
      }
    ],
    "stream": false
  }'
```
