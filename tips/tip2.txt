Here’s the practical list **as of May 6, 2026**. I’m treating “connections/hour” as **API inference requests/hour**. Where a provider publishes only RPM, I compute **theoretical RPH = RPM × 60**, but daily caps often bind first.

## Best overall picks

**Best raw coding power:** Google **Gemini 2.5 Pro** and Mistral **Codestral/Devstral**, but both use dashboard/header-specific free limits rather than a stable public RPM table. Google says active limits must be checked in AI Studio and are enforced by RPM/TPM/RPD. Mistral says Experiment/free limits are limited and visible in Admin/headers. ([Google AI for Developers][1])

**Best transparent free bandwidth:** **Cerebras** and **Groq**. Cerebras publishes free-plan RPM/RPH/RPD tables, and Groq publishes free-plan RPM/RPD/TPM/TPD by model. ([Cerebras Inference][2])

**Highest nominal free RPM:** **Cloudflare Workers AI** at 300 RPM for text generation, but the free quota is **10,000 neurons/day**, so long coding calls will hit the daily free quota before they hit RPM. ([Cloudflare Docs][3])

# Setup + API call examples

Use environment variables and never ship API keys in frontend code.

## 1. Google Gemini API

Setup: go to **Google AI Studio → Get API key**, create a key, set `GEMINI_API_KEY`. Google says free-tier rate limits are project/model/tier-specific and visible in AI Studio. ([Google AI for Developers][1])

```bash
export GEMINI_API_KEY="YOUR_KEY"

curl -X POST "https://generativelanguage.googleapis.com/v1beta/interactions" \
  -H "Content-Type: application/json" \
  -H "x-goog-api-key: $GEMINI_API_KEY" \
  -d '{
    "model": "gemini-2.5-pro",
    "input": "Write a robust TypeScript function to topologically sort test jobs by dependency."
  }'
```

Google’s Interactions API docs show the REST endpoint pattern and supported Gemini model IDs, including `gemini-2.5-pro`. ([Google AI for Developers][17])

---

## 2. Cerebras

Setup: create a free Cerebras API key in Cerebras Cloud, set `CEREBRAS_API_KEY`. Cerebras docs explicitly say you can get started with a free API key. ([Cerebras Inference][18])

```bash
export CEREBRAS_API_KEY="YOUR_KEY"

curl https://api.cerebras.ai/v1/chat/completions \
  -H "Authorization: Bearer $CEREBRAS_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-oss-120b",
    "messages": [
      {"role": "user", "content": "Review this Rust function for lifetime bugs and propose a fix."}
    ]
  }'
```

Cerebras uses Bearer auth and the `/v1/chat/completions` endpoint. ([Cerebras Inference][19])

---

## 3. Groq

Setup: create a key in **Groq Console → API Keys**, set `GROQ_API_KEY`.

```bash
export GROQ_API_KEY="YOUR_KEY"

curl https://api.groq.com/openai/v1/chat/completions \
  -H "Authorization: Bearer $GROQ_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "qwen/qwen3-32b",
    "messages": [
      {"role": "user", "content": "Generate Jest tests for a React hook that debounces API calls."}
    ]
  }'
```

Groq documents OpenAI-compatible chat completion endpoints, and publishes free-plan limits per model. ([GroqCloud][20])

---

## 4. Cloudflare Workers AI

Setup: in Cloudflare Dashboard, open **Workers AI → Use REST API**, create a Workers AI API token, and copy your Account ID. ([Cloudflare Docs][21])

```bash
export CLOUDFLARE_ACCOUNT_ID="YOUR_ACCOUNT_ID"
export CLOUDFLARE_AUTH_TOKEN="YOUR_TOKEN"

curl "https://api.cloudflare.com/client/v4/accounts/$CLOUDFLARE_ACCOUNT_ID/ai/run/@cf/qwen/qwen2.5-coder-32b-instruct" \
  -X POST \
  -H "Authorization: Bearer $CLOUDFLARE_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {"role": "user", "content": "Write a Vite + React security checklist for XSS and unsafe dependencies."}
    ]
  }'
```

Cloudflare’s Qwen2.5-Coder page lists the model ID and REST examples. ([Cloudflare Docs][8])

## 5. Mistral

Setup: create a Mistral account, create an API key in the console, set `MISTRAL_API_KEY`. For coding, start with `codestral-latest`; Mistral also exposes code/FIM endpoints. ([Mistral AI][5])

```bash
export MISTRAL_API_KEY="YOUR_KEY"

curl https://api.mistral.ai/v1/chat/completions \
  -H "Authorization: Bearer $MISTRAL_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "codestral-latest",
    "messages": [
      {"role": "user", "content": "Refactor this Python class into a clean repository/service pattern."}
    ]
  }'
```

Mistral’s rate limits are organization-level and exposed through Admin/headers rather than a fixed public free RPM table. ([Mistral AI][12])

## 6. GitHub Models

Setup: open GitHub Models, create a GitHub personal access token with `models:read`, set `GITHUB_TOKEN`. GitHub’s quickstart shows this workflow and the `https://models.github.ai/inference/chat/completions` endpoint. ([GitHub Docs][9])

```bash
export GITHUB_TOKEN="YOUR_GITHUB_PAT"

curl -L https://models.github.ai/inference/chat/completions \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4.1",
    "messages": [
      {"role": "user", "content": "Explain how to migrate this Node.js CommonJS module to ESM safely."}
    ]
  }'
```

GitHub’s free limits vary by model category; high-tier models are generally much more capped than low-tier ones. ([GitHub Docs][13])

## 7. OpenRouter

Setup: create an OpenRouter account, create an API key, then choose a model ending in `:free` from the model catalog. OpenRouter’s free variants are limited to 20 RPM, with 50/day or 1,000/day depending on purchased credits. ([OpenRouter][10])

```bash
export OPENROUTER_API_KEY="YOUR_KEY"

curl https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "REPLACE_WITH_A_CURRENT_MODEL_ID_ENDING_IN_:free",
    "messages": [
      {"role": "user", "content": "Find bugs in this TypeScript async queue implementation."}
    ]
  }'
```

OpenRouter is OpenAI-chat-compatible, but exact free model IDs change frequently. ([OpenRouter][22])

## 8. Hugging Face Inference Providers

Setup: create a Hugging Face token with permission to make Inference Provider calls. HF says free users get a small monthly credit allocation routed through Hugging Face. ([Hugging Face][23])

```bash
export HF_TOKEN="YOUR_HF_TOKEN"

curl https://router.huggingface.co/v1/chat/completions \
  -H "Authorization: Bearer $HF_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-ai/DeepSeek-V3-0324",
    "messages": [
      {"role": "user", "content": "Write a Go parser for a simple .env file format."}
    ]
  }'
```

This is best for experimentation, not heavy free coding traffic, because the free monthly credit is small.

## My recommended stack

For **serious free coding work**, I’d use this order:

1. **Gemini 2.5 Pro** for hardest coding, architecture, long-context repo reasoning.
2. **Cerebras `gpt-oss-120b` or `qwen-3-235b-a22b`** for high-volume free calls.
3. **Groq `qwen/qwen3-32b`** for fast iterative code generation.
4. **Cloudflare `@cf/qwen/qwen2.5-coder-32b-instruct`** for code-specific calls where 10K neurons/day is enough.
5. **Mistral `codestral-latest`** for code completion/FIM and targeted coding tasks.
6. **OpenRouter `:free` models** as a fallback aggregator when a specific free route is temporarily good.

The two most important operational rules: check the provider’s live dashboard/headers before building anything automated, and rotate across providers by task type rather than hammering one free quota.
