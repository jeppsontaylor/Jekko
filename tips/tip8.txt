As of **May 6, 2026**, these are the **top 20 free hosted API routes for coding-capable LLMs** I could verify from current provider docs/pages. I’m ranking by **coding strength first**, then by **usable free API bandwidth**.

A few important notes:

“RPH” below means **requests per hour**. When a provider publishes only RPM, I show **RPM × 60 as burst RPH**, but daily/token/credit caps may stop you earlier. Google does **not** publish one stable free RPM/RPH number for each Gemini model; their docs say active limits are shown in AI Studio and are measured by RPM, TPM, and RPD. OpenRouter’s pure free daily cap is only **50 requests/day**, even though the burst rate is 20 RPM. Cloudflare’s free tier is usage-metered by **10,000 Neurons/day**, so the 300 RPM platform cap is not the only limiter.

## Best picks by use case

For **maximum coding quality**, start with:

1. `gemini-2.5-pro`
2. `@cf/moonshotai/kimi-k2.6`
3. `gemini-3-flash-preview`
4. `minimax/minimax-m2.5:free`
5. `openai/gpt-4.1` via GitHub Models

For **most free API calls**, start with:

1. Cloudflare Workers AI routes, especially `@cf/qwen/qwen2.5-coder-32b-instruct`, `@cf/openai/gpt-oss-120b`, `@cf/moonshotai/kimi-k2.6`, subject to the **10,000 Neurons/day** budget.
2. Cerebras `gpt-oss-120b` or `qwen-3-235b-a22b-instruct-2507`, because **900 RPH / 14,400 RPD** is very generous for a free hosted API.
3. Groq `openai/gpt-oss-120b` or `qwen/qwen3-32b`, if you want simple OpenAI-compatible calls and fast inference.

For **purely free with no paid credits attached**, be careful with OpenRouter: the useful burst limit is **20 RPM**, but the pure free daily cap is **50 requests/day**. OpenRouter raises free-model daily usage to **1,000/day** only after at least $10 of purchased credits. ([OpenRouter][18])

# API key setup + example calls

## 1. Google AI Studio / Gemini API

**Setup URL:** [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey)

**Steps**

1. Go to [https://aistudio.google.com/apikey](https://aistudio.google.com/apikey)
2. Sign in with Google.
3. Click **Create API key**.
4. Select or create a Google Cloud project.
5. Copy the key.
6. Set it locally:

```bash
export GEMINI_API_KEY="YOUR_KEY_HERE"
```

Google’s docs say Gemini API keys are created and managed in Google AI Studio. ([Google AI for Developers][19])

**Python example**

```bash
pip install -U google-genai
export GEMINI_API_KEY="YOUR_KEY_HERE"
```

```python
from google import genai

client = genai.Client()

response = client.models.generate_content(
    model="gemini-2.5-pro",
    contents=(
        "You are a senior software engineer. "
        "Write a Python LRU cache implementation with type hints and pytest tests."
    ),
)

print(response.text)
```

To use Gemini 3 Flash Preview:

```python
model="gemini-3-flash-preview"
```

To use Gemini 2.5 Flash:

```python
model="gemini-2.5-flash"
```

## 2. OpenRouter

**Setup URL:** [https://openrouter.ai/keys](https://openrouter.ai/keys)

**Steps**

1. Go to [https://openrouter.ai/keys](https://openrouter.ai/keys)
2. Sign in.
3. Click **Create API key**.
4. Copy the key.
5. Set it locally:

```bash
export OPENROUTER_API_KEY="YOUR_KEY_HERE"
```

OpenRouter’s docs show OpenAI-compatible `/chat/completions` usage, and their limits page documents the free-model caps. ([OpenRouter][18])

**cURL example**

```bash
export OPENROUTER_API_KEY="YOUR_KEY_HERE"
export MODEL="minimax/minimax-m2.5:free"

curl https://openrouter.ai/api/v1/chat/completions \
  -H "Authorization: Bearer $OPENROUTER_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "'"$MODEL"'",
    "messages": [
      {
        "role": "system",
        "content": "You are an expert coding assistant. Return correct, production-quality code."
      },
      {
        "role": "user",
        "content": "Write a TypeScript topological sort function that detects cycles. Include tests."
      }
    ],
    "temperature": 0.2
  }'
```

Swap in any OpenRouter free model from the table:

```bash
export MODEL="qwen/qwen3-coder:free"
export MODEL="nvidia/nemotron-3-super-120b-a12b:free"
export MODEL="tencent/hy3-preview:free"
export MODEL="inclusionai/ling-2.6-1t:free"
```

**Check your OpenRouter key limits**

```bash
curl https://openrouter.ai/api/v1/key \
  -H "Authorization: Bearer $OPENROUTER_API_KEY"
```

## 4. GitHub Models

**Setup URLs:**

API token: [https://github.com/settings/tokens](https://github.com/settings/tokens)
Model catalog: [https://github.com/marketplace/models](https://github.com/marketplace/models)

**Steps**

1. Go to [https://github.com/settings/tokens](https://github.com/settings/tokens)
2. Create a personal access token.
3. Give it the `models:read` permission.
4. Save the token.
5. Set it locally:

```bash
export GITHUB_TOKEN="YOUR_GITHUB_PAT"
```

GitHub documents free API usage for GitHub Models, PAT setup with `models:read`, and model-tier rate limits. ([GitHub Docs][21])

**cURL example**

```bash
export GITHUB_TOKEN="YOUR_GITHUB_PAT"

curl -L https://models.github.ai/inference/chat/completions \
  -H "Accept: application/vnd.github+json" \
  -H "Authorization: Bearer $GITHUB_TOKEN" \
  -H "X-GitHub-Api-Version: 2026-03-10" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "openai/gpt-4.1",
    "messages": [
      {
        "role": "system",
        "content": "You are a senior backend engineer."
      },
      {
        "role": "user",
        "content": "Refactor this Express.js route for validation, typed errors, and unit-testable structure."
      }
    ],
    "temperature": 0.2
  }'
```

## 7. Cloudflare Workers AI

**Setup URLs:**

API tokens: [https://dash.cloudflare.com/profile/api-tokens](https://dash.cloudflare.com/profile/api-tokens)
Workers AI dashboard: [https://dash.cloudflare.com/](https://dash.cloudflare.com/)

**Steps**

1. Go to [https://dash.cloudflare.com/profile/api-tokens](https://dash.cloudflare.com/profile/api-tokens)
2. Create a token with Workers AI permissions.
3. Copy your **Account ID** from the Cloudflare dashboard.
4. Set both locally:

```bash
export CLOUDFLARE_ACCOUNT_ID="YOUR_ACCOUNT_ID"
export CLOUDFLARE_AUTH_TOKEN="YOUR_API_TOKEN"
```

Cloudflare’s REST API docs say you need an API token and Account ID, and Cloudflare provides an OpenAI-compatible chat-completions endpoint. ([Cloudflare Docs][23])

**OpenAI-compatible cURL example**

```bash
export CLOUDFLARE_ACCOUNT_ID="YOUR_ACCOUNT_ID"
export CLOUDFLARE_AUTH_TOKEN="YOUR_API_TOKEN"
export MODEL="@cf/moonshotai/kimi-k2.6"

curl "https://api.cloudflare.com/client/v4/accounts/$CLOUDFLARE_ACCOUNT_ID/ai/v1/chat/completions" \
  -H "Authorization: Bearer $CLOUDFLARE_AUTH_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "'"$MODEL"'",
    "messages": [
      {
        "role": "system",
        "content": "You are an expert coding assistant."
      },
      {
        "role": "user",
        "content": "Review this Python function for bugs and return corrected code with tests."

    ],
    "temperature": 0.2
  }'
```

Use this combination for the best free coding workflow:

**Primary quality model:** `gemini-2.5-pro`

**OpenRouter temporary power picks:** `minimax/minimax-m2.5:free`, `qwen/qwen3-coder:free`, and any temporary `:free` frontier routes while they last.
