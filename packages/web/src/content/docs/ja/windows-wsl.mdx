Jekko は Windows で直接実行できますが、より快適に使うには [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) の利用をおすすめします。WSL は Jekko の機能とスムーズに連携する Linux 環境を提供します。

:::tip[WSL を使う理由]
WSL を使うと、ファイルシステム性能、ターミナルサポート、Jekko が依存する開発ツールとの互換性が向上します。
:::

---

## セットアップ

<Steps>

1. **WSL をインストールする**

   まだの場合は、Microsoft 公式ガイドを使って [WSL をインストール](https://learn.microsoft.com/en-us/windows/wsl/install) します。

2. **WSL で Jekko をインストールする**

   WSL の準備ができたら WSL のターミナルを開き、[インストール方法](/docs/) のいずれかで Jekko をインストールします。

   ```bash
   curl -fsSL https://jekko.ai/install | bash
   ```

3. **WSL から Jekko を使う**

   プロジェクトディレクトリに移動し（Windows ファイルは `/mnt/c/` や `/mnt/d/` などからアクセス）、Jekko を実行します。

   ```bash
   cd /mnt/c/Users/YourName/project
   jekko
   ```

</Steps>

## デスクトップアプリ + WSL サーバー

Jekko デスクトップアプリを使いつつ、サーバーは WSL で動かしたい場合は次の手順です。

1. **WSL でサーバーを起動する**

   外部接続を許可するため、`--hostname 0.0.0.0` を付けて起動します。

   ```bash
   jekko serve --hostname 0.0.0.0 --port 4096
   ```

2. **デスクトップアプリを接続する**

   `http://localhost:4096` に接続します。

:::note
環境によって `localhost` が使えない場合は、WSL 側で `hostname -I` を実行して IP アドレスを確認し、`http://<wsl-ip>:4096` に接続してください。
:::
:::caution
`--hostname 0.0.0.0` を使う場合は、`JEKKO_SERVER_PASSWORD` を設定してサーバーを保護してください。

```bash
JEKKO_SERVER_PASSWORD=your-password jekko serve --hostname 0.0.0.0
```

:::

## Web クライアント + WSL

Windows で Web 利用を快適にするには:

1. **PowerShell ではなく WSL ターミナルで `jekko web` を実行する**

   ```bash
   jekko web --hostname 0.0.0.0
   ```

2. **Windows のブラウザからアクセスする**

   `http://localhost:<port>` にアクセスします（URL は Jekko が表示します）。

WSL から `jekko web` を実行すると、適切なファイルシステムアクセスとターミナル統合を維持したまま、Windows ブラウザから利用できます。

---

## Windows ファイルへのアクセス

WSL からは `/mnt/` ディレクトリ経由で Windows ファイルにアクセスできます。

- `C:` drive → `/mnt/c/`
- `D:` drive → `/mnt/d/`
- そのほかのドライブも同様です

例:

```bash
cd /mnt/c/Users/YourName/Documents/project
jekko
```

:::tip
よりスムーズに使うには、リポジトリを WSL のファイルシステム（例: `~/code/`）にクローンまたはコピーして、そこで Jekko を実行することをおすすめします。
:::

---

## ヒント

- Windows ドライブ上のプロジェクトでも、Jekko は WSL で実行するとファイルアクセスがスムーズです
- Jekko と一緒に VS Code の [WSL 拡張機能](https://code.visualstudio.com/docs/remote/wsl) を使うと統合的な開発フローを構築できます
- Jekko の設定とセッションは WSL 環境内の `~/.local/share/jekko/` に保存されます
